--- 2. СТРУКТУРА МОНОРЕПО ---
Корень: barter-clone (private, workspaces: apps/*, packages/*)

npm scripts (корень package.json):
  dev       → concurrently: веб + API (то же, что dev:all)
  dev:all   → concurrently: dev:web + dev:api
  dev:web   → только Next, через scripts/npm.ps1 -w @app/web dev
  dev:api   → Nest watch, -w @app/api start:dev
  build     → build во всех workspace

Если нужен только фронт — используйте dev:web при уже запущенном API.

--- 10. ТИПИЧНЫЕ ПРИЧИНЫ «НЕ ЗАПУСКАЕТСЯ / НЕ РАБОТАЕТ 3000» ---
1) Запущен только Next, не запущен Nest на 3001 → SSR fetch падает (ошибка сети в apiGetJson).
2) PostgreSQL не запущен или неверный DATABASE_URL → Nest падает при первом запросе к Prisma или при старте модулей.
3) Порт 3000 или 3001 занят другим процессом.
4) Meilisearch: при недоступном сервере после правок API не должен падать; если старая версия кода — проверить meiliReady.
5) Windows: предпочтительно открывать http://127.0.0.1:3000 (согласовано с API_URL).
