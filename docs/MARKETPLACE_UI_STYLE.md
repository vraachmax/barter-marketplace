# Стиль UI «Бартер» — маркетплейс (западные площадки)

**Канонический документ.** При любой новой верстке/редизайне в `apps/web` опираться на этот файл и на `.cursor/rules/marketplace-ui-style.mdc`.

**Референсы по духу:** eBay, Facebook Marketplace, Vinted, Etsy — чистая иерархия, плотные карточки, заметный primary CTA, аккуратный dark mode.

---

## 1. Бренд и акценты

| Роль | Светлая тема | Тёмная тема |
|------|----------------|-------------|
| **Primary / CTA** | Градиент `from-sky-600 to-cyan-600`, текст белый, тень `shadow-sky-600/25`–`/30` | Те же градиенты или чуть ярче hover; кнопки «Найти» на цветной полосе можно `cyan-500` + тёмный текст |
| **Акцент ссылки** | `text-sky-700`, hover `sky-800`, подчёркивание при необходимости | `text-sky-400`, hover `sky-300` |
| **Нейтральный фон страницы** | `bg-zinc-100` | `bg-zinc-950` |
| **Карточки / панели** | `bg-white`, граница `border-zinc-200` | `bg-zinc-950` / `zinc-900/80`, `border-zinc-700`–`800` |
| **Вторичный акцент** | Архив / особые вкладки: **фиолетовый** (`violet-*`) | Сохранить читаемый контраст |
| **Успех / verified** | `emerald-*` | `emerald` с `dark:` вариантами |
| **Опасность / unread** | `rose` / `red` дозированно | Не «кислотный» на тёмном |

Полоса поиска (бренд): класс **`.brand-search-shell`** в `globals.css` — градиент sky/cyan; в dark — приглушённый teal/cyan.

---

## 2. Скругления (важно)

- **Корпус крупных плавающих панелей / модалок / дропдаунов:** `rounded-3xl`, часто `overflow-hidden`.
- **Карточки секций, сайдбары, крупные блоки:** `rounded-2xl`.
- **Кнопки шапки, инпуты, строки списков, мелкие плашки:** `rounded-xl` или `rounded-2xl` в зависимости от плотности.
- **Пузыри исходящих сообщений / pill CTA:** `rounded-2xl` или `rounded-3xl`.
- **Иконки в квадрате** в меню (маркетплейс-строки): `grid h-9 w-9` … `rounded-xl`.

Избегать «плоских» `rounded-md` на новых экранах для основных контейнеров.

---

## 3. Тени и глубина

- Карточки: `shadow-md` / `shadow-lg`, в dark — `dark:shadow-black/20`–`/50`.
- CTA: лёгкая цветная тень от градиента (`shadow-sky-600/25`).
- Sticky header: `backdrop-blur`, полупрозрачный фон, `border-b`.

---

## 4. Типографика и иерархия

- Заголовки страниц: `font-bold` / `font-black`, плотный `tracking-tight` где уместно.
- Подзаголовки: `text-sm text-zinc-500` (dark: `zinc-400`).
- Метки секций: `text-[11px] font-bold uppercase tracking-wide` для «каталог / сводка» и т.п.

---

## 5. Тёмная тема (`html[data-theme="dark"]`)

- Tailwind-префикс **`dark:`** синхронизирован с **`data-theme`** (см. `@custom-variant dark` в `globals.css`), а не только с `prefers-color-scheme`.
- Классы вида **`bg-white/95`** не переопределяются глобальным `.bg-white` — для полей на цветной полосе и шапки задавать явные **`dark:bg-slate-900/95`**, **`dark:text-zinc-100`**, **`dark:border-*`**.
- Шапка приложения должна быть **тёмной целиком** в dark (не оставлять светлый бар с нечитаемым логотипом).
- Z-index: **шапка и её дропдауны** выше плавающих виджетов (напр. чат **`z-30`**, header **`z-50`**).

---

## 6. Паттерны компонентов

- **Герой:** градиентный фон (sky/cyan + нейтраль), бейдж с иконкой (напр. Sparkles), CTA — градиент sky→cyan.
- **Сайдбар категорий:** шапка с градиентом; активный пункт — sky-фон + ring; иконки в квадрате 9×9.
- **Выпадающее меню профиля:** внешний `rounded-3xl`, шапка с градиентом, пункты — строка с цветной иконкой в `rounded-xl`; без дублей ссылок, которые уже в шапке (на главной — «На главную» вместо сообщений/подачи, если дублируется).
- **Виджет чатов:** `rounded-3xl` корпус, внутренняя шапка `rounded-2xl`, отступ снизу на мобилке под bottom nav (`max-md:bottom-28`).
- **Кабинет продавца:** справа сверху — **«На главную»** вместо дублей «Сообщения» / «Новое объявление» (чаты и подача — в сайдбаре и KPI).

---

## 7. Глобальные CSS

Файл **`apps/web/src/app/globals.css`**:

- `.brand-search-shell`, `.hero-banner`, `.hero-banner-title`
- `.listing-thumb-wrap`, `.listing-thumb-img`, `.listing-thumb-shade`, `.listing-placeholder-surface`
- Совместимость dark для legacy-классов (`bg-white`, `text-zinc-*`, …)

Новые экраны: по возможности **явные `dark:`** в разметке, не полагаться только на overrides.

---

## 8. Иконки

- Набор: **lucide-react**, обводка согласованная (`strokeWidth={2}` на крупных кнопках по желанию).
- Иконка в primary-кнопке часто белая на градиенте.

---

## 9. Чеклист перед сдачей UI

- [ ] Скругления согласованы с уровнем блока (2xl / 3xl).
- [ ] Primary действия — sky→cyan; ссылки — sky.
- [ ] Dark: фон карточек/инпутов не «белые пятна» на тёмной странице.
- [ ] Нет лишних дублей навигации в одном контексте.
- [ ] Z-index: поповеры шапки не перекрываются виджетами.
- [ ] Обновить таблицу в `HANDOFF.md` (раздел UI), если добавлен новый крупный экран.
